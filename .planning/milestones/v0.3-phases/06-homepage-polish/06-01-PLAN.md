  <action>
1. In `components/ui/skeleton.tsx`, change `bg-accent` to `bg-muted` in the className string. The line should read:
   `className={cn("bg-muted animate-pulse rounded-md", className)}`
   This is the ONLY change needed — all skeleton instances across the app use this base `Skeleton` component, so every loading skeleton (home, discover, series, library, recommendations) inherits the fix automatically.

2. In `app/globals.css`, add a custom `animate-pulse` override INSIDE the `@layer base` block to slow down the default Tailwind pulse animation. Add this after the existing body rule:

   ```css
   @keyframes pulse {
     0%, 100% {
       opacity: 1;
     }
     50% {
       opacity: 0.4;
     }
   }
   ```

   Then add a utility override after the `@layer base` block:

   ```css
   .animate-pulse {
     animation: pulse 3s cubic-bezier(0.4, 0, 0.6, 1) infinite;
   }
   ```

   This changes the default 2s pulse to 3s, making it noticeably calmer. The opacity range (1 to 0.4) is slightly more subtle than Tailwind's default (1 to 0.5).

No other files need changes. The `bg-accent` references in `button.tsx`, `toggle.tsx`, `dialog.tsx`, `dropdown-menu.tsx`, `badge.tsx` are correct interactive hover/focus states — NOT skeleton colors — and must NOT be changed.
  </action>
