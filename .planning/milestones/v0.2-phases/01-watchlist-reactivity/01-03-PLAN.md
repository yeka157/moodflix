### Task 2: Animation Polish — Icon Transitions & Rating Bounce

**File(s):** `components/movies/movie-card.tsx`, `components/movies/movie-detail-modal.tsx`, `components/watchlist/watchlist-card.tsx`

**Action:** Add Framer Motion micro-animations for icon state changes and rating interactions per CONTEXT.md feedback decisions.

**Details:**

**Icon state transitions (movie card + modal):**
- Wrap bookmark and check icons in `motion.div` or use `motion` on the button
- `whileTap={{ scale: 0.85 }}` on all icon buttons for tactile feedback
- When icon transitions from outline to filled (adding to library), use:
  ```typescript
  <motion.div
    key={`bookmark-${status}`}  // key change triggers animation
    initial={{ scale: 0.5, opacity: 0 }}
    animate={{ scale: 1, opacity: 1 }}
    transition={{ type: "spring", stiffness: 500, damping: 15 }}
  >
  ```
- Respect `prefers-reduced-motion`: wrap animations in `motion-safe:` Tailwind variants for CSS transitions, and use `useReducedMotion()` from Framer Motion for JS animations

**Like/dislike bounce (all locations — modal + library card):**
- Add `whileTap={{ scale: 0.85 }}` to thumb buttons
- When rating changes (fill color appears), animate with:
  ```typescript
  <motion.div
    key={`rating-${item.rating}`}
    initial={{ scale: 1.3 }}
    animate={{ scale: 1 }}
    transition={{ type: "spring", stiffness: 400, damping: 10 }}
  >
  ```
- Color transition handled via CSS `transition-colors duration-200` (already exists via `cn()`)

**Important:** Do NOT add animations that block interaction. All animations are cosmetic overlays on instant state changes. The optimistic update (Plan 01) drives the state; animations are decorative.

**Verify:**
- `npm run build` passes
- `npm run lint` passes
- Clicking bookmark/check icons shows scale bounce
- Filling an icon (adding) shows spring-in animation
- Like/dislike shows color change with subtle bounce
- With `prefers-reduced-motion: reduce` enabled in OS settings, animations are suppressed
- No animation blocks or delays user interaction
