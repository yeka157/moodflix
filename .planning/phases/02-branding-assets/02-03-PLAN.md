**Step 1 — Download Bebas Neue TTF:**

The font is needed by Satori (the engine behind `next/og` ImageResponse). It cannot use `next/font/google` CSS variables — it needs the raw TTF binary read from disk at request time.

```bash
mkdir -p /Users/kevin/Repository/moodflix/assets
curl -L "https://fonts.gstatic.com/s/bebasneue/v14/JTUSjIg69CK48gW7PXoo9WdhyyTh89ZNpQ.woff2" \
  -o /Users/kevin/Repository/moodflix/assets/BebasNeue-Regular.woff2
```

Wait — Satori needs a TTF/OTF buffer, not woff2. Download the TTF directly from Google Fonts static CDN:

```bash
curl -L "https://fonts.gstatic.com/s/bebasneue/v14/JTUSjIg69CK48gW7PXoo9WdhyyTh89ZNpQ.ttf" \
  -o /Users/kevin/Repository/moodflix/assets/BebasNeue-Regular.ttf 2>/dev/null
```

If the above URL doesn't resolve the TTF (Google Fonts may serve woff2 for that URL), use this alternative approach — download the CSS first to extract the actual TTF source URL:

```bash
# Get the font CSS to find the correct TTF URL
curl -A "Mozilla/5.0" "https://fonts.googleapis.com/css2?family=Bebas+Neue&display=swap" 2>/dev/null | head -20
```

Or use npm to fetch via google-fonts-complete:

```bash
# Alternative: use the fontsource package
npm install --save-dev @fontsource/bebas-neue
cp node_modules/@fontsource/bebas-neue/files/bebas-neue-latin-400-normal.woff2 assets/BebasNeue-Regular.woff2
npm uninstall @fontsource/bebas-neue
```

Note: Satori actually supports woff2 in addition to TTF/OTF. Either format works. Use whichever successfully downloads. Name the file `BebasNeue-Regular.ttf` (or `BebasNeue-Regular.woff2` and update the OG image code accordingly).

**Simplest reliable approach:** Download the woff2 from fontsource which is a known reliable CDN:
```bash
mkdir -p /Users/kevin/Repository/moodflix/assets
curl -L "https://cdn.jsdelivr.net/fontsource/fonts/bebas-neue@latest/latin-400-normal.woff2" \
  -o /Users/kevin/Repository/moodflix/assets/BebasNeue-Regular.ttf
```

The file extension is `.ttf` for consistency with the code below, even if the binary is woff2 — Satori reads the format from the buffer magic bytes, not the extension.

Verify the file is non-empty:
```bash
ls -la /Users/kevin/Repository/moodflix/assets/BebasNeue-Regular.ttf
```

**Step 2 — Create `app/opengraph-image.tsx` (global OG):**

CRITICAL Satori rules (violations produce silent failures or broken images):
- NO `display: grid` — use `display: flex` only
- NO `filter: blur()` — silently dropped (GitHub issue #573)
- NO React component functions inside ImageResponse JSX — inline all SVG/HTML as flat elements
- NO `oklch()` colors — use `#FB2C36` hex
- All elements need explicit `display: flex` if they contain children
- `position: absolute` works but requires parent to have `position: relative` and explicit `display: flex`

The background uses an abstract cinematic grid of colored rectangles (simulating blurred movie poster slots) — this is the approved alternative to a blurred poster collage. No real movie images, no copyright issues, no blur required.

```tsx
import { ImageResponse } from "next/og";
import { readFile } from "node:fs/promises";
import { join } from "node:path";

export const alt = "Moodflix - AI-Powered Movie Discovery";
export const size = { width: 1200, height: 630 };
export const contentType = "image/png";

// Cinematic grid color palette — muted film poster tones
const GRID_COLORS = [
  "#1a0a0a", "#0a0f1a", "#0d1a0a", "#1a150a",
  "#0a0a1a", "#1a0a15", "#0f1a1a", "#1a1a0a",
  "#140a1a", "#0a1a14", "#1a0f0a", "#0a141a",
];

export default async function Image() {
  const fontData = await readFile(
    join(process.cwd(), "assets/BebasNeue-Regular.ttf")
  );

  return new ImageResponse(
    (
      <div
        style={{
          width: "100%",
          height: "100%",
          display: "flex",
          alignItems: "center",
          justifyContent: "center",
          backgroundColor: "#0a0a0a",
          position: "relative",
        }}
      >
        {/* Abstract cinematic grid background — 4 columns × 3 rows of colored film poster slots */}
        <div
          style={{
            position: "absolute",
            inset: 0,
            display: "flex",
            flexWrap: "wrap",
            opacity: 0.4,
          }}
        >
          {GRID_COLORS.map((color, i) => (
            <div
              key={i}
              style={{
                width: "25%",
                height: "33.333%",
                backgroundColor: color,
                borderRight: "2px solid #0a0a0a",
                borderBottom: "2px solid #0a0a0a",
                display: "flex",
              }}
            />
          ))}
        </div>

        {/* Dark vignette gradient overlay */}
        <div
          style={{
            position: "absolute",
            inset: 0,
            background:
              "linear-gradient(135deg, rgba(10,10,10,0.90) 0%, rgba(10,10,10,0.65) 50%, rgba(10,10,10,0.90) 100%)",
            display: "flex",
          }}
        />

        {/* Centered wordmark */}
        <div
          style={{
            position: "relative",
            display: "flex",
            flexDirection: "column",
            alignItems: "center",
            justifyContent: "center",
          }}
        >
          {/* MOODFLIX — crimson M + white OODFLIX */}
          <div
            style={{
              display: "flex",
              fontSize: 130,
              fontFamily: "BebasNeue",
              letterSpacing: "-3px",
              lineHeight: 1,
            }}
          >
            <span style={{ color: "#FB2C36" }}>M</span>
            <span style={{ color: "#ffffff" }}>OODFLIX</span>
          </div>

          {/* Thin crimson rule below wordmark */}
          <div
            style={{
              width: 80,
              height: 3,
              backgroundColor: "#FB2C36",
              marginTop: 16,
              display: "flex",
            }}
          />
        </div>
      </div>
    ),
    {
      ...size,
      fonts: [
        {
          name: "BebasNeue",
          data: fontData,
          style: "normal",
          weight: 400,
        },
      ],
    }
  );
}
```

Note: The global OG image has NO subtitle — just the wordmark with the decorative rule. Subtitles appear only in the route-specific variants (Task 2).
