<task type="auto">
  <name>Task 2: Consolidate AI logging into onFinish callback + add off-topic logging</name>
  <files>app/api/ai/recommend/route.ts</files>
  <action>
Refactor `app/api/ai/recommend/route.ts` to consolidate all conversation logging:

**Step 1: Add a `buildConversationMetadata` helper function** (local to route.ts) that takes `event.steps` and extracts:
- `toolUsed`: "suggest_genres" | "identify_media" | "none"
- `genresSuggested`: string[] (genre names from suggest_genres args)
- `mediaIdentified`: string[] (titles from identify_media results)
- `messageCount`: number (uiMessages.length + 1 for the new AI response)
- `isOffTopic`: boolean (always false here; true only on off-topic path)

Inspect `step.toolCalls` for tool names and args, `step.toolResults` for identify_media results. Cast metadata as `Record<string, unknown>` for the Drizzle insert.

**Step 2: Add `onFinish` callback to the `streamText` call.** Inside onFinish:
1. Call `buildConversationMetadata(event.steps)` to derive metadata
2. Add `messageCount: uiMessages.length + 1` to the metadata object
3. Build the full conversation array: `[...uiMessages, { role: "assistant", parts: [{ type: "text", text: event.text }] }]`
4. Upsert into `aiConversations` using the existing conversationId pattern:
   - If `conversationId` exists: insert with `onConflictDoUpdate` targeting `aiConversations.conversationId`, updating `messages`, `prompt`, `updatedAt`, and `metadata`
   - If no `conversationId`: plain insert
5. All DB operations are fire-and-forget (`.catch(() => {})`)

Note: `onFinish` fires AFTER the stream completes and AFTER the response is already returned to the client. Fire-and-forget is natural here.

**Step 3: Remove the logging code from inside `suggest_genres.execute` and `identify_media.execute`.** The `db.insert(aiConversations)` blocks in both tool execute functions should be deleted entirely. Keep the `db.insert(aiRecommendations)` in suggest_genres (that's a separate table for recommendation tracking, not conversation logging).

**Step 4: Add off-topic conversation logging.** Before the existing `return createUIMessageStreamResponse(...)` in the off-topic early-return path:
1. Build the off-topic conversation array: `[...uiMessages, { role: "assistant", parts: [{ type: "text", text: message }] }]`
2. Build off-topic metadata: `{ toolUsed: "none", isOffTopic: true, messageCount: offTopicMessages.length }`
3. Upsert into `aiConversations` using the same conversationId pattern as onFinish
4. Fire-and-forget (`.catch(() => {})`)

This ensures off-topic redirects (which bypass `streamText` entirely) are also logged.

**Important details:**
- The `uiMessages` and `lastMessageText` variables are already in scope from the route handler
- The `conversationId` variable is already parsed from the request body
- Cast metadata as `Record<string, unknown>` to satisfy Drizzle's JSONB typing
- Do NOT change the `suggest_genres.execute` return value or `identify_media.execute` return value — only remove the conversation logging from them
