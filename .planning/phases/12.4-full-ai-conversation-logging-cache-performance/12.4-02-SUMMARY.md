# Phase 12.4 Plan 02: SSR Prefetch & HydrationBoundary for Discover and Series Summary

**TanStack Query SSR prefetching added to /discover and /series pages, eliminating loading skeleton flash on first visit and enabling stale-while-revalidate on return navigation**

## Accomplishments

- /discover page now prefetches default discover infinite query server-side using `discoverMovies()` from `@/lib/tmdb`, seeding TanStack Query cache before the page renders
- /series page seeds TanStack Query cache via `setQueryData` for all 5 TV categories (trending, top_rated, korean_drama, chinese_drama, airing_today) from already-fetched SSR data
- Both pages wrapped in `HydrationBoundary` so client components pick up SSR-prefetched data without re-fetching on first visit
- Exported `getDefaultDiscoverParams()` helper from `hooks/use-movies.ts` for exact query key alignment between server prefetch and client `useDiscoverMovies` hook

## Files Created/Modified

- `app/(app)/discover/page.tsx` - Converted to async Server Component; prefetchInfiniteQuery with movieKeys.discover(defaultParams); HydrationBoundary wrapper
- `app/(app)/series/page.tsx` - Added QueryClient + setQueryData for 5 TV categories after existing Promise.all; HydrationBoundary wrapper
- `hooks/use-movies.ts` - Added `getDefaultDiscoverParams()` export returning `{ genreId: "", sortBy: "popularity.desc", year: "" }`
- `app/api/ai/recommend/route.ts` - Pre-existing TypeScript error fix (dead `buildConversationMetadata` + unused `StepResult`/`ToolSet` imports)

## Decisions Made

- Used `discoverMovies()` from `@/lib/tmdb` directly in the Server Component prefetch queryFn — client-side `fetchDiscover` uses relative `/api/movies` URLs that fail on the server
- Used `setQueryData` (not `prefetchQuery`) for /series because data is already fetched in the Promise.all; avoids a second identical fetch
- Created `new QueryClient()` per request (not `makeQueryClient()`) per TanStack Query SSR docs — Server Components must never share a singleton QueryClient
- `dehydrate(queryClient)` called after all prefetches/setQueryData complete
