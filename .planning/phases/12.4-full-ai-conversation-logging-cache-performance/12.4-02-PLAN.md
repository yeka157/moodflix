**Discover page (`app/(app)/discover/page.tsx`):**

Convert to an async Server Component (it currently is not async). Add SSR prefetching:

1. Import `dehydrate`, `HydrationBoundary`, `QueryClient` from `@tanstack/react-query`
2. Import `movieKeys` from `@/hooks/use-movies` (safe â€” plain object, not a hook)
3. Import TMDB cache functions: `getCachedTrending`, `getCachedPopular`, `getCachedTopRated` from `@/lib/tmdb-cache` (or the appropriate server-side fetch functions)
4. Create a fresh `QueryClient` (NOT `makeQueryClient()` â€” see research pitfall 5)
5. The DiscoverGridContent component uses `useDiscoverMovies` with an infinite query. The default params are `{genreId: "", sortBy: "popularity.desc", year: ""}`. Use `prefetchInfiniteQuery`:

```typescript
const defaultParams = { genreId: "", sortBy: "popularity.desc", year: "" };

await queryClient.prefetchInfiniteQuery({
  queryKey: movieKeys.discover(defaultParams),
  queryFn: async () => {
    // Fetch page 1 of discover with default sort (popularity.desc)
    // This matches what /api/movies?action=discover&page=1 returns
    const res = await fetch(
      `${process.env.NEXT_PUBLIC_SUPABASE_URL ? "" : ""}...`
    );
    // Actually, use the TMDB cache or direct TMDB call server-side
  },
  initialPageParam: 1,
});
```

**IMPORTANT query key alignment:** The client `useDiscoverMovies` uses `movieKeys.discover(params)` as the query key for an infinite query. The server must prefetch with the EXACT same key and infinite query shape. Use `queryClient.prefetchInfiniteQuery` (not `prefetchQuery`) with `initialPageParam: 1`.

For the queryFn, since we're on the server, call the TMDB discover endpoint directly using the existing server-side TMDB client. Import `discoverMovies` from `@/lib/tmdb` (or the equivalent function that `/api/movies` route calls internally). If no such function exists, use `getCachedPopular(1)` as a close approximation â€” but the query key MUST match exactly.

**Alternative approach (simpler, recommended):** Instead of prefetching the infinite query (which has complex serialization), prefetch the individual category queries that the page would show if we added category rows. BUT DiscoverGridContent doesn't use category queries â€” it uses `useDiscoverMovies` infinite query exclusively.

**Simplest correct approach:** Since DiscoverGridContent uses `useDiscoverMovies` (infinite query), and the default params produce a fetch to `/api/movies?action=discover&page=1&sort_by=popularity.desc`, we need to replicate that server-side. The `/api/movies` route calls `discoverByGenre` or a discover endpoint from TMDB. Look at `app/api/movies/route.ts` to find the exact TMDB function called for `action=discover` and use it directly in the prefetch queryFn.

The response format from the API must match `MovieListResponse` exactly â€” the same shape the client queryFn returns. If there's a format mismatch, the hydration will fail silently (data won't be picked up).

6. Wrap `<DiscoverGridContent />` in `<HydrationBoundary state={dehydrate(queryClient)}>`.
7. Keep the existing metadata export and page markup unchanged.
