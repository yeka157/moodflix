**Create `app/api/tv/route.ts`** — Mirror the structure of `app/api/movies/route.ts`:

1. Import `NextRequest` from `next/server`
2. Import `getTrendingTV`, `getTopRatedTV`, `discoverKoreanDramas`, `discoverChineseDramas` from `@/lib/tmdb`
3. Export `GET` handler that:
   - Extracts `category` and `page` from `request.nextUrl.searchParams`
   - Defaults `page` to 1 via `Number(searchParams.get("page") ?? "1")`
   - Switches on `category`:
     - `"trending"` → `Response.json(await getTrendingTV(page))`
     - `"top_rated"` → `Response.json(await getTopRatedTV(page))`
     - `"korean_drama"` → `Response.json(await discoverKoreanDramas(page))`
     - `"chinese_drama"` → `Response.json(await discoverChineseDramas(page))`
     - `default` → `Response.json({ error: "Provide valid 'category' parameter" }, { status: 400 })`
   - Wraps in try/catch: catch returns `Response.json({ error: "Failed to fetch TV shows" }, { status: 500 })`

This route does NOT support `query` or `genre` params (no TV search or genre discover in Phase 4). Keep it simple — 4 categories only.

**Create `app/api/tv/[id]/route.ts`** — Mirror the structure of `app/api/movies/[id]/route.ts`:

1. Import `NextRequest` from `next/server`
2. Import `getTVDetails` from `@/lib/tmdb`
3. Import `getCountryFromHeaders` from `@/lib/country`
4. Export `GET` handler with signature: `(request: NextRequest, { params }: { params: Promise<{ id: string }> })`
5. Handler:
   - `const { id } = await params` (Next.js 16 async params)
   - Validate: `Number.isNaN(Number(id))` → return 400 `"Invalid TV show ID"`
   - `const country = getCountryFromHeaders(request.headers)`
   - `const details = await getTVDetails(Number(id))`
   - Extract providers: `details["watch/providers"]?.results?.[country] || null`
   - Return `Response.json({ ...details, watchProviders: providers, watchCountry: country, mediaType: "tv" as const })`
   - Wrap in try/catch: catch returns 500 `"Failed to fetch TV details"`

The `mediaType: "tv"` discriminant is critical — Phase 5's modal will use it to branch rendering.

**Query key factory:**
```typescript
export const tvKeys = {
  all: ["tv"] as const,
  category: (cat: TVCategory) => [...tvKeys.all, "category", cat] as const,
  details: (id: number) => [...tvKeys.all, "details", id] as const,
};
```

**Category type:**
```typescript
export type TVCategory = "trending" | "top_rated" | "korean_drama" | "chinese_drama";
```

**Shared fetch function (private):**
```typescript
async function fetchTVCategory(category: TVCategory): Promise<Movie[]> {
  const res = await fetch(`/api/tv?category=${category}`);
  if (!res.ok) throw new Error("Failed to fetch TV shows");
  const data: TVListResponse = await res.json();
  return data.results.map(normalizeTVShow);  // Normalize at hook boundary per locked decision
}
```

This is where normalization happens — `normalizeTVShow()` is called inside `queryFn`, NOT at the API route level. This is a locked decision.

**Category hooks (4 hooks):**
- `useTrendingTV()` — `useQuery({ queryKey: tvKeys.category("trending"), queryFn: () => fetchTVCategory("trending"), staleTime: 10 * 60 * 1000, gcTime: 30 * 60 * 1000 })`
- `useTopRatedTV()` — same pattern, `staleTime: 30 * 60 * 1000, gcTime: 60 * 60 * 1000` (top rated changes less frequently)
- `useKoreanDramas()` — `staleTime: 10 * 60 * 1000, gcTime: 30 * 60 * 1000`
- `useChineseDramas()` — `staleTime: 10 * 60 * 1000, gcTime: 30 * 60 * 1000`

**TV details fetch function (private):**
```typescript
async function fetchTVDetails(id: number): Promise<TVDetailsResponse> {
  const res = await fetch(`/api/tv/${id}`);
  if (!res.ok) throw new Error("Failed to fetch TV details");
  return res.json();
}
```

Note: `fetchTVDetails` does NOT normalize — it returns `TVDetailsResponse` as-is because the detail modal needs TV-specific fields (seasons, episodes, created_by, status). Only list hooks normalize to `Movie[]`.

**Details hook:**
- `useTVDetails(id: number | null)` — `useQuery({ queryKey: tvKeys.details(id!), queryFn: () => fetchTVDetails(id!), enabled: id !== null, staleTime: 30 * 60 * 1000, gcTime: 60 * 60 * 1000 })`

All hooks return TanStack Query result objects. The 4 category hooks return `UseQueryResult<Movie[], Error>`. The details hook returns `UseQueryResult<TVDetailsResponse, Error>`.
