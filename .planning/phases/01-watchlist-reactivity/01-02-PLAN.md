### Task 2: Movie Card — Dual Icon System (Bookmark + Check)

**File(s):** `components/movies/movie-card.tsx`

**Action:** Replace the single bookmark icon with two stacked icons: bookmark (want to watch) and check (watched). Show both on hover, only the active one when not hovering. Each icon acts as a one-tap action.

**Details:**

**Import changes:**
- Add `CircleCheck` from lucide-react (alongside existing `Bookmark`)
- Import `useRemoveFromWatchlist`, `useUpdateWatchlistStatus` from hooks
- Import `WatchlistTmdbEntry` type
- Remove `toast` import for success cases (icon animation is the feedback per CONTEXT.md decisions)
- Keep `toast` for error cases only

**Data layer changes:**
- `useWatchlistTmdbIds()` now returns `WatchlistTmdbEntry[]`
- Derive status: `const watchlistEntry = watchlistEntries?.find(e => e.tmdbId === movie.id)`
- `const status = watchlistEntry?.status ?? null` (null = not in library)
- Replace `isInWatchlist` boolean with status check

**Icon behavior (per CONTEXT.md decisions):**

| Current State | Bookmark Icon | Check Icon |
|---|---|---|
| Not in library | Outline bookmark, click → add as `want_to_watch` | Outline check, click → add as `watched` |
| `want_to_watch` | Filled bookmark (crimson), click → remove from library | Outline check, click → change to `watched` |
| `watched` | Outline bookmark, click → change to `want_to_watch` | Filled check (green), click → remove from library |

**Layout:**
- Two icons stacked vertically in top-right corner (bookmark on top, check below)
- Both visible on hover; only the active icon visible when not hovering
- If no active state, both hidden until hover
- Each icon is 32x32px (h-8 w-8) with rounded-full background
- Gap of 2 (gap-2) between icons

**Icon styling:**
- Inactive: `bg-black/60 hover:bg-black/80 text-white` (existing style)
- Active bookmark: `bg-primary text-primary-foreground` with `fill-current` on the Bookmark icon
- Active check: `bg-green-600 text-white` with `fill-current` on the CircleCheck icon

**Click handlers:**
- Bookmark click: If not in library → `addMutation.mutate({ status: "want_to_watch" })`. If want_to_watch → `removeMutation.mutate()`. If watched → `statusMutation.mutate({ status: "want_to_watch" })`.
- Check click: If not in library → `addMutation.mutate({ status: "watched" })`. If watched → `removeMutation.mutate()`. If want_to_watch → `statusMutation.mutate({ status: "watched" })`.

**Feedback (per CONTEXT.md):**
- Success: Icon animation only (Framer Motion scale bounce: `whileTap={{ scale: 0.85 }}` + fill transition via CSS)
- No success toasts for add/toggle
- Error: `toast.error()` with rollback (handled by hook's `onError`)
- Remove: `toast("Removed from library", { action: { label: "Undo", onClick: () => addMutation.mutate(...) }, duration: 5000 })` — only when clicking active icon to remove

**Note:** The remove and status-change actions need the watchlist item `id` (UUID). Plan 01 Task 2 already includes `id: string` in `WatchlistTmdbEntry`, so `watchlistEntry.id` is available directly — no extra queries needed.

**Verify:**
- `npm run build` passes
- Movie cards show bookmark icon for want_to_watch items, check icon for watched items
- Both icons appear on hover for any movie card
- Clicking bookmark on untracked movie adds as want_to_watch
- Clicking check on untracked movie adds as watched
- No success toasts on add/toggle (only icon visual change)
- Remove action shows undo toast
