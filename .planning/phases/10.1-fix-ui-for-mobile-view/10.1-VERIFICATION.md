## Notes

**movie-grid.tsx reduced-motion handling:** The `MovieGrid` component uses `whileInView` without an explicit `useReducedMotion()` call. This is acceptable — Framer Motion v10+ automatically skips `initial`/`animate`/`whileInView` transitions when `prefers-reduced-motion: reduce` is set at the OS level. The plan's instruction to keep existing reduced-motion checks in place was satisfied (no checks were removed). The marquee in `movie-showcase.tsx` uses `motion-safe:` / `motion-reduce:` CSS variants explicitly, which is correct for CSS animations.
