<task type="auto">
  <name>Task 1: Tune Framer Motion spring configs for smoother animations</name>
  <files>
    components/movies/movie-card.tsx

    components/watchlist/watchlist-card.tsx
  </files>
  <action>
Audit and fix rigid Framer Motion spring configs across the app. The goal is natural, fluid motion — one clean overshoot or direct settle, not rapid oscillations.

**movie-card.tsx (4 changes):**

1. Card hover scale (2 occurrences, lines ~332-333 and ~344-345):
   - Change `whileHover={{ scale: 1.05 }}` to `whileHover={{ scale: 1.04 }}`
   - Change `transition={{ type: "spring", stiffness: 300, damping: 20 }}` to `transition={{ type: "spring", stiffness: 200, damping: 25 }}`
   - WHY: stiffness 300 / damping 20 produces 2-3 oscillations. 200/25 produces one clean settle.

2. Bookmark icon appear (line ~228):
   - Change `transition={{ type: "spring", stiffness: 500, damping: 15 }}` to `transition={{ type: "spring", stiffness: 300, damping: 22 }}`
   - WHY: 500/15 is extremely snappy with 3-4 oscillations — looks mechanical, especially on mobile where tap latency makes it jarring.

3. CircleCheck icon appear (line ~273):
   - Same change: `stiffness: 500, damping: 15` to `stiffness: 300, damping: 22`

4. If `whileTap={{ scale: 0.85 }}` exists on small icon buttons (< 44px), soften to `scale: 0.92` for a gentler tap feel. Leave `scale: 0.85` on larger CTA buttons.

**movie-grid.tsx (1 change):**

1. Grid item whileInView entry (line ~45):
   - Change `transition={{ duration: 0.4 }}` to `transition={{ duration: 0.45, ease: [0.25, 0.1, 0.25, 1] }}`
   - WHY: The default ease is close to linear — adding a cubic-bezier eases items in more naturally.
   - Also change `initial={{ opacity: 0, y: 20 }}` to `initial={{ opacity: 0, y: 16 }}` — shorter travel distance feels less dramatic.

**movie-detail-modal.tsx (2 changes):**

1. Like icon spring (line ~602):
   - Change `transition={{ type: "spring", stiffness: 400, damping: 10 }}` to `transition={{ type: "spring", stiffness: 300, damping: 20 }}`
   - WHY: 400/10 bounces excessively (4+ oscillations). 300/20 gives one clean pop.

2. Dislike icon spring (line ~634):
   - Same change: `stiffness: 400, damping: 10` to `stiffness: 300, damping: 20`

**watchlist-card.tsx (1 change):**

1. Rating icon animation (line ~122):
   - Change `stiffness: 400, damping: 10` to `stiffness: 300, damping: 20`
   - Same reasoning: reduce excessive bounce.

Do NOT change:
- `app-sidebar.tsx` spring configs (stiffness: 280, damping: 26 — already well-tuned)
- `install-prompt.tsx` spring configs (stiffness: 300, damping: 30 — already good)
- Any scroll-driven `useTransform` animations in hero-section.tsx — those are intentionally linear to scroll position
- The `prefers-reduced-motion` checks — they must stay in place
