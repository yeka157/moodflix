<task type="auto">
  <name>Task 2: Fix marquee fallback duplicate poster rows</name>
  <files>components/landing/movie-showcase.tsx</files>
  <action>
Fix the MarqueeFallback component that shows two identical rows of the same movies on mobile.

1. **Split posters into two distinct halves (line ~74-76):**
   Replace:
   ```tsx
   const row1 = [...posters, ...posters];
   const row2 = [...posters, ...posters];
   ```
   With:
   ```tsx
   const half = Math.ceil(posters.length / 2);
   const firstHalf = posters.slice(0, half);
   const secondHalf = posters.length >= 4
     ? posters.slice(half)
     : [...posters].reverse();
   // Each row still doubles for seamless CSS marquee loop
   const row1 = [...firstHalf, ...firstHalf];
   const row2 = [...secondHalf, ...secondHalf];
   ```

2. **Verify row keys are unique:**
   - row1 uses `key={row1-${index}}` and row2 uses `key={row2-${index}}` — these are already distinct, no change needed.

3. **Edge case:** If `posters.length < 4`, the secondHalf would be very short. The fallback `[...posters].reverse()` ensures the second row still has content but in reversed order (visually distinct from row 1).

Do NOT change:
- The CSS marquee animation keyframes (in globals.css)
- The marquee-left / marquee-right direction assignments
- The gradient fade overlays on left/right edges
- The PosterCard component
  </action>
