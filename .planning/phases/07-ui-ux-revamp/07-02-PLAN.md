**1. Create `app/(app)/movie/[id]/page.tsx`** (Server Component):
- `params` is `Promise<{ id: string }>` in Next.js 16 — must await it
- Parse `id` to number, call `notFound()` if NaN
- Fetch in parallel: `getMovieDetails(movieId)` + `getMovieRecommendations(movieId)` using `Promise.all`
- Extract regional watch providers: `details["watch/providers"]?.results?.[country] ?? null`
- Get country from `getCountryFromHeaders(await headers())`
- Render `<MovieDetailPageContent>` passing `details`, `watchProviders`, `recommendations`
- Add `generateMetadata` for SEO: title = movie title, description = overview truncated to 160 chars, openGraph image = TMDB backdrop URL

**2. Create `components/movies/movie-detail-page.tsx`** ("use client"):
- Props: `{ details: MovieDetails & extras, watchProviders: WatchProvider | null, recommendations: Movie[] }`
- **Full-bleed backdrop section**: TMDB backdrop image (w1280) filling viewport width, `aspect-[21/9]` or `h-[50vh]` with gradient fade to background at bottom. Title overlaid at bottom-left.
- **Metadata row**: Pill badges using `<Badge variant="secondary">` for:
  - Runtime (e.g. "2h 15m" — convert `details.runtime` minutes)
  - Year (from `release_date`)
  - Rating (e.g. "7.8 ★" from `vote_average`)
  - Certification if available
- **Genre tags**: Row of pill badges for each genre, using `<Badge variant="outline">`
- **Overview section**: `details.overview` text
- **Director**: Extract from `details.credits.crew` where `job === "Director"`, display name(s)
- **Cast chips**: Horizontal scrollable row of pill-style `<Badge>` with actor names (first 10-15 from `details.credits.cast`). No photos — just name chips (per Stremio reference).
- **Watch providers section**: If providers exist, show streaming/rent/buy sections with small provider logos (same pattern as existing modal, reuse `PROVIDER_URLS` from constants)
- **"More Like This" row**: Use `<MovieRow>` component with `recommendations` array. Each card should link to `/movie/{id}` (will be wired in Plan 04).
- **Fixed bottom action bar**: `position: fixed bottom-0` (above mobile tab bar — use `bottom-16 md:bottom-0`), full width, `bg-background/80 backdrop-blur-sm border-t`. Contains:
  - "Add to Library" button (or "In Library" if already added) — uses `useWatchlistTmdbIds` + `useAddToWatchlist` / `useRemoveFromWatchlist`
  - "Watched" toggle button — uses `useUpdateWatchlistStatus`
  - Like / Dislike buttons — uses existing rating hooks
  - Trailer button (links to YouTube search if no direct trailer URL, or uses TMDB video data if available)
- Use Framer Motion for entrance animations: backdrop scale, staggered content reveal (per existing hero pattern)
- Respect `useReducedMotion`

**3. Create `app/(app)/movie/[id]/loading.tsx`:**
- Skeleton matching the detail page layout: backdrop placeholder (aspect-[21/9] bg-muted), title skeleton, pill badge skeletons, overview text skeletons, cast chip skeletons
- Use existing `Skeleton` component from shadcn/ui
