**conversationId-based upsert so multi-turn AI chats produce one DB row per session instead of duplicates**

## Accomplishments
- Multi-turn AI conversations now produce a single database row per session, updated on each suggest_genres call
- Client-generated conversationId (UUID) ties all messages in a chat session to one row
- Fire-and-forget pattern fully preserved -- no latency impact on streaming responses
- Backward-compatible: old clients without conversationId fall back to plain insert

## Files Created/Modified

- `drizzle/migrations/0003_safe_shiva.sql` - Migration: ALTER TABLE add columns + CREATE UNIQUE INDEX with WHERE clause

- `app/api/ai/recommend/route.ts` - Accept optional conversationId, upsert with onConflictDoUpdate when present

## Decisions Made
- Used `useState(() => crypto.randomUUID())` instead of `useRef(...).current` because React 19 ESLint rules forbid accessing ref.current during render
- Added `updatedAt` column (not in original schema) to track when multi-turn conversations were last updated -- needed for the upsert set clause
- Partial unique index (`WHERE conversation_id IS NOT NULL`) instead of regular unique constraint -- allows existing rows without conversationId to coexist
