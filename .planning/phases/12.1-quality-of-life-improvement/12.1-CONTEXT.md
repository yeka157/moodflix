# Phase 12.1: Quality of Life Improvement - Context

**Gathered:** 2026-03-07
**Status:** Ready for planning

<domain>
## Phase Boundary

Quick UI/UX fixes and conversation logging improvements. Four items:
1. Fix AI chat send button icon (replace "AI" icon with standard send icon)
2. Fix mobile bottom tab bar — no active indicator on detail pages (/movie/[id], /tv/[id])
3. Add back navigation on movie/TV detail pages for mobile
4. Log full multi-turn AI conversations to database (schema adjustment + route update)

</domain>

<decisions>
## Implementation Decisions

### AI Chat Send Button
- Replace the current "AI" icon on the send button with a standard send icon (e.g., ArrowUp or Send from lucide-react)
- Should look intuitive — users expect a send/arrow icon, not a branded "AI" label

### Mobile Bottom Tab Bar on Detail Pages
- Detail pages (/movie/[id], /tv/[id]) currently show no active state on the bottom tab bar
- These pages don't belong to a specific tab, but the bar should still provide visual context
- Options: highlight the tab the user came from, or show no highlight but keep the bar visible and functional

### Back Navigation on Detail Pages (Mobile)
- Currently no way to go back on mobile detail pages — swiping refreshes instead of navigating back
- Add a back button on the detail page for mobile users

### Conversation Logging
- Full multi-turn conversation should be logged to the `ai_conversations` table, not just the initial exchange
- Currently only the first genre suggestion logs the conversation — subsequent turns in the same chat session are lost
- Schema may need adjustment: current `messages` column stores a single conversation snapshot; needs to capture the full multi-turn history
- Log only, no UI — chat still starts fresh for the user each time, no viewable history
- Fire-and-forget pattern must be preserved (no latency impact on streaming)

### Claude's Discretion
- Back button style (floating overlay vs header bar — pick what fits the existing cinematic detail page design)
- Bottom tab bar behavior on detail pages (dim all tabs, highlight parent tab, or other pattern)
- Schema adjustment approach for multi-turn logging (update existing column vs new rows per turn)
- Send button icon choice (ArrowUp vs Send vs other lucide icon)

</decisions>

<specifics>
## Specific Ideas

- User specifically mentioned: "when I swipe, I thought it was like a drawer, but when I swipe it, it refreshes the page" — the detail page feels like it should be dismissible but isn't
- The bottom bar "doesn't have any indication for anything selected" on detail pages — it's confusing to not know where you are
- Send button: user said the "AI" icon is "not intuitive" — standard messaging convention is an arrow/send icon

</specifics>
