<objective>
Improve AI conversation logging so multi-turn chats produce a single database row per session instead of one row per genre suggestion. Add a client-generated conversationId to enable upsert behavior.

Purpose: Currently each `suggest_genres` tool call inserts a new row. In multi-turn conversations, this creates redundant rows where later rows are supersets of earlier ones. Using a conversationId with upsert ensures one canonical row per conversation, always containing the full message history.

Output: Updated use-ai.ts hook with conversationId, updated AI route with upsert logic.
</objective>
