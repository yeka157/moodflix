## Summary

This phase adds a "Trending TV Shows" `MovieRow` to the homepage and a third "Browse Series" feature card. All required infrastructure already exists: `getTrendingTV()` is in `lib/tmdb.ts`, `normalizeTVShow()` is in `types/tv.ts`, `MovieRow` accepts `mediaType="tv"` and routes to `/tv/[id]`, and the `/tv/[id]` detail pages exist. The work is pure composition — no new files, no new API routes, no schema changes.

The homepage currently fetches `getTrendingMovies()` and `getPersonalizedData()` in a `Promise.all`, then conditionally fetches `getPopularMoviesInRegion()`. The TV fetch should be added to the first `Promise.all` to run in parallel with the existing movie fetch, minimising added latency. The fetched TV data flows through the Server Component (`home/page.tsx`) to the Client Component (`HomeMovies`) as a prop.

The only discretionary question from the user is whether to add a regional popular TV row. Research confirms TMDB does support `/tv/popular` with a `region` parameter, but the user's `getPopularMoviesInRegion()` is movie-only and there is no equivalent TV function in the codebase yet. Given the phase goal is "one TV row is enough to surface TV content on the homepage" (from CONTEXT.md Specifics), the regional TV row should NOT be added in this phase — it would require writing a new TMDB function and complicating the fetch logic with little benefit beyond the trending row.

**Primary recommendation:** Add `getTrendingTV()` to the existing `Promise.all` in `home/page.tsx`, pass the normalized results to `HomeMovies`, render a `MovieRow` with `mediaType="tv"` after the movie trending row, and add `"Tv"` to the `iconMap` in `feature-card-grid.tsx` alongside a new feature card definition. Four targeted edits, zero new files.

## Architecture Patterns

### Recommended Project Structure

No new files needed. All changes are edits to existing files:

```
app/(app)/home/page.tsx          # Add getTrendingTV to Promise.all, pass trendingTV prop
components/movies/home-movies.tsx # Accept trendingTV prop, render TV MovieRow
components/layout/feature-card-grid.tsx # Add "Tv" to iconMap
app/(app)/home/page.tsx          # Add "Browse Series" feature card to featureCards array
```

### Pattern 1: Parallel Server Fetch + Prop Drilling

**What:** Fetch TV data in the same `Promise.all` as movies in the Server Component, then pass the normalized array down as a prop to the Client Component.

**When to use:** Always when two fetches are independent — they have no data dependency on each other.

**Example (current pattern to extend):**
```typescript
// app/(app)/home/page.tsx — current
const [trending, personalizedData] = await Promise.all([
  getTrendingMovies(),
  user ? getPersonalizedData(user.id) : Promise.resolve(null),
]);

// Extended pattern for this phase
const [trending, trendingTV, personalizedData] = await Promise.all([
  getTrendingMovies(),
  getTrendingTV().catch(() => ({ results: [], page: 1, total_pages: 0, total_results: 0 })),
  user ? getPersonalizedData(user.id) : Promise.resolve(null),
]);

const trendingTVMovies = trendingTV.results.map(normalizeTVShow);
```

**Why `.catch()` on TV fetch:** The `/series` page uses `.catch()` guards on all TV fetches to ensure a single TMDB API failure doesn't break the entire page. Apply the same pattern here for resilience.

### Pattern 2: iconMap Extension in FeatureCardGrid

**What:** `FeatureCardGrid` maps string icon names to Lucide components via a Record lookup. Adding a new icon requires two steps: import the icon, add it to the map.

**When to use:** Any time a new feature card is added.

**Example:**
```typescript
// components/layout/feature-card-grid.tsx — current
import { ArrowRight, Compass, Bookmark, Sparkles } from "lucide-react";

const iconMap: Record<string, LucideIcon> = {
  Compass,
  Bookmark,
  Sparkles,
};

// Extended for this phase
import { ArrowRight, Compass, Bookmark, Sparkles, Tv } from "lucide-react";

const iconMap: Record<string, LucideIcon> = {
  Compass,
  Bookmark,
  Sparkles,
  Tv,
};
```

### Pattern 3: HomeMovies Prop Extension

**What:** `HomeMovies` is a Client Component that accepts typed props from the Server Component. Adding the TV row means adding a `trendingTV` prop of type `Movie[]`.

**When to use:** When adding new data rows that originate from server-side fetches.

**Example:**
```typescript
// components/movies/home-movies.tsx — extended interface
interface HomeMoviesProps {
  trending: Movie[];
  trendingTV: Movie[];         // NEW
  personalizedData?: PersonalizedData | null;
  regionalPopular?: Movie[];
}

// Usage in JSX (after existing Trending Movies row):
<MovieRow
  title="Trending TV Shows"
  movies={trendingTV}
  mediaType="tv"
/>
```

### Anti-Patterns to Avoid

- **Using `hrefPrefix` as a URL prefix string:** `MovieRow` uses `resolvedHrefPrefix` only as a boolean to decide whether to render links. The actual href is always `/${movie.media_type ?? mediaType}/${movie.id}`. Do NOT attempt to set `hrefPrefix="/tv/"` expecting it to be used literally as the URL prefix — it will be ignored. Use `mediaType="tv"` instead (since `normalizeTVShow` does not set `media_type` on the returned Movie object, the `mediaType` prop is the fallback that correctly routes to `/tv/[id]`).
- **Setting `hrefPrefix` redundantly:** In `SeriesContent`, `hrefPrefix="/tv/"` is set but ignored due to the above. For the homepage TV row, just pass `mediaType="tv"` — no need for `hrefPrefix` because there is no `onMovieClick` handler (direct navigation links are desired).
- **Adding a regional TV row in this phase:** No `getPopularTVInRegion()` exists. Adding it would require a new TMDB function, extra TMDB API call, and added complexity. The phase goal explicitly says one TV row is enough.
- **Awaiting `getTrendingTV()` sequentially:** Never `await` it after the existing `Promise.all` — this serialises two independent network calls and doubles perceived load time.
- **Passing unnormalized `TVShow[]` to `MovieRow`:** `MovieRow` accepts `Movie[]`. Always call `.map(normalizeTVShow)` on the TMDB TV response before passing to components.

## Common Pitfalls

### Pitfall 1: MovieRow href construction misunderstanding

**What goes wrong:** Developer passes `hrefPrefix="/tv/"` expecting it to be used as the URL base, but gets the same href as if `mediaType="tv"` was used.

**Why it happens:** `MovieRow` lines 35 and 130 show that `hrefPrefix` is used as a truthy check (sets `resolvedHrefPrefix`) but the actual constructed href is always `/${movie.media_type ?? mediaType}/${movie.id}`. The string value of `hrefPrefix` is never interpolated into the URL.

**How to avoid:** Use `mediaType="tv"` without `hrefPrefix`. Since `normalizeTVShow` returns a Movie without `media_type` set, the `mediaType` prop is the fallback — passing `mediaType="tv"` produces `/tv/${movie.id}` links correctly.

**Warning signs:** If you see `hrefPrefix="/tv/"` on a row that also has `mediaType="tv"`, the `hrefPrefix` is redundant noise (not harmful, but unnecessary).

### Pitfall 2: TypeScript error — missing `trendingTV` prop

**What goes wrong:** After adding `trendingTV` to `HomeMoviesProps`, the call site in `home/page.tsx` will get a TypeScript error if the prop is not passed, because it's required.

**Why it happens:** Required props in TypeScript interfaces cause build-time errors if not provided.

**How to avoid:** Both changes (interface + call site) must be made in the same task. Either make `trendingTV` optional with `trendingTV?: Movie[]` and default to `[]` inside the component, or update both files atomically. Optional with default is cleaner since it makes the component more resilient.

### Pitfall 3: Build fail — `Tv` icon not in iconMap

**What goes wrong:** Feature card with `icon: "Tv"` renders without an icon (falls back to `Compass`) if `"Tv"` is not added to `iconMap` in `feature-card-grid.tsx`.

**Why it happens:** The map lookup `iconMap[card.icon] ?? Compass` silently falls back. No TypeScript error, no runtime error — just wrong icon.

**How to avoid:** Add `Tv` import and `Tv` key to `iconMap` in the same commit as the feature card data change. Verify in browser after implementation.

### Pitfall 4: TV fetch failure breaking homepage

**What goes wrong:** If TMDB `/trending/tv/week` returns an error (5xx, rate limit), the `Promise.all` throws, crashing `home/page.tsx` rendering.

**Why it happens:** Unguarded `Promise.all` fails entirely when any member rejects.

**How to avoid:** Apply `.catch(() => ({ results: [], page: 1, total_pages: 0, total_results: 0 }))` to the `getTrendingTV()` call — same pattern used by `/series` page. Then guard the row: `{trendingTV.length > 0 && <MovieRow ... />}` so an empty array silently hides the row rather than rendering an empty row component.

### Pitfall 5: Server → Client component serialization of normalized array

**What goes wrong:** Passing a complex object through a Server → Client component boundary can fail if the object contains non-serializable values (functions, class instances, Dates).

**Why it happens:** Next.js serializes Server Component props to JSON for Client Components.

**How to avoid:** `normalizeTVShow` returns a plain `Movie` object with only primitive fields (number, string, boolean, null, number[]). This is safe. No special handling needed.

### Full diff pattern for home-movies.tsx

```typescript
// components/movies/home-movies.tsx
interface HomeMoviesProps {
  trending: Movie[];
  trendingTV?: Movie[];   // optional — gracefully renders nothing if empty/absent
  personalizedData?: PersonalizedData | null;
  regionalPopular?: Movie[];
}

export function HomeMovies({
  trending,
  trendingTV = [],
  personalizedData,
  regionalPopular,
}: HomeMoviesProps) {
  return (
    <>
      <MovieRow
        title="Trending This Week"
        movies={trending}
        mediaType="movie"
      />

      {trendingTV.length > 0 && (
        <MovieRow
          title="Trending TV Shows"
          movies={trendingTV}
          mediaType="tv"
        />
      )}

      {personalizedData && <PersonalizedSection data={personalizedData} />}

      {!personalizedData && regionalPopular && regionalPopular.length > 0 && (
        <MovieRow
          title="Popular in Your Region"
          movies={regionalPopular}
          mediaType="movie"
        />
      )}
    </>
  );
}
```
