# Phase 09.1.1: Homepage TV Series Integration - Context

**Gathered:** 2026-03-02
**Status:** Ready for planning

<domain>
## Phase Boundary

Add TV series content to the homepage, which currently only shows movies (trending, personalized/regional popular). The `/series` page and TMDB TV functions already exist — this phase brings TV visibility to the homepage. No new pages, no new API routes, no watchlist changes.

</domain>

<decisions>
## Implementation Decisions

### Content mix
- Separate rows for movies and TV — not mixed together
- Add a "Trending TV Shows" row using existing `getTrendingTV()` from `lib/tmdb.ts`
- Normalize TV data with existing `normalizeTVShow()` so `MovieRow` can render them
- TV rows link to `/tv/[id]` detail pages (already exist)

### Row placement
- Homepage order: Hero Banner → Welcome → Trending Movies → Trending TV Shows → (Personalized or Regional Popular) → Mood Section → Feature Cards
- TV row sits directly below the existing movie trending row — keeps movies first but surfaces TV early
- Hero banner stays movie-only (it uses `Movie` type and links to movie detail)

### Personalized section
- Personalized recommendations stay movie-only for now — the `getPersonalizedData` function queries watchlist by movie TMDB IDs and uses `getMovieDetails`
- Adding TV to personalized section would require significant refactoring of the recommendation logic — that's a separate effort

### Feature cards
- Add a third feature card linking to `/series` — "Browse Series" alongside existing "Discover Movies" and "My Library"
- Icon: `Tv` from lucide-react

### Claude's Discretion
- Exact data fetching pattern (parallel with existing fetches vs separate)
- Whether to add a "Popular TV in Your Region" row (depends on TMDB API availability for regional TV)
- Loading/error handling for the TV row
- MovieRow props configuration for TV content

</decisions>

<specifics>
## Specific Ideas

- Keep it simple — one TV row is enough to surface TV content on the homepage
- The series page already has trending, Korean dramas, Chinese dramas, top rated — homepage just needs a teaser
- Use the same `MovieRow` component with `mediaType="tv"` — no new components needed

</specifics>

<deferred>
## Deferred Ideas

- TV content in personalized recommendations — requires refactoring `getPersonalizedData` to handle mixed media types
- TV shows in hero banner rotation — would need `HeroBanner` to accept TV shows and link to `/tv/[id]`
- "Continue Watching" row for TV series with episode progress — future feature (TVFUT-01)

</deferred>
