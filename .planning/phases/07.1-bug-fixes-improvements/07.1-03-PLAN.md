    2. Create `actions/profile.ts`:
       - Server action `updateUsername(username: string)`:
         - Get user via `supabase.auth.getUser()` — return `{ error: "Not authenticated" }` if no user.
         - Validate username with Zod: min 2, max 30 chars, regex `/^[a-zA-Z0-9_-]+$/`, allow empty string (to clear).
         - If validation fails, return `{ error: "..." }` with the Zod error message.
         - Use Drizzle to update: `db.update(profiles).set({ username: username || null }).where(eq(profiles.id, user.id))`.
         - Import `profiles` from `@/drizzle/schema` and `eq` from `drizzle-orm`.
         - Return `{ success: true }` on success.
         - Wrap in try/catch, return `{ error: "Failed to update username" }` on DB error.
         - Call `revalidatePath("/settings")` after successful update.

    3. Create `app/(app)/settings/page.tsx` (Server Component):
       - Fetch user: `const supabase = await createClient(); const { data: { user } } = await supabase.auth.getUser();`
       - If no user, redirect to `/login`.
       - Fetch profile from Drizzle: `const profile = await db.select().from(profiles).where(eq(profiles.id, user.id)).limit(1);`
       - Extract display name priority: `profile[0]?.username ?? user.user_metadata?.full_name ?? user.email?.split("@")[0] ?? "User"`
       - Extract avatar URL from Google OAuth: `user.user_metadata?.avatar_url as string | null ?? null`
       - Render page with heading "Settings", profile info section showing avatar (if available), display name, email, and the `<SettingsForm>` component.
       - Pass `currentUsername`, `displayName`, `email`, `avatarUrl` as props to SettingsForm.
       - Use `import type { Metadata } from "next"` and export `metadata = { title: "Settings" }`.

    4. Create `components/settings/settings-form.tsx` ("use client"):
       - Use `react-hook-form` with `zodResolver` for the username field.
       - Zod schema: `z.object({ username: z.string().max(30).regex(/^[a-zA-Z0-9_-]*$/, "...").optional().or(z.literal("")) })`.
       - Form shows:
         - Profile section at top: Avatar (using `next/image` if `avatarUrl` exists, otherwise `Avatar` + `AvatarFallback` with initial), display name, email (read-only).
         - Username input field with label "Username", placeholder "Enter a username", current value pre-filled.
         - Save button with loading state (use `useTransition` for the server action call).
         - Success/error feedback via `sonner` toast.
       - On submit: call `updateUsername(data.username)` server action, show toast on result.
       - Style with existing shadcn/ui components: Card, Input, Button, Label, Avatar.

    5. Update `next.config.ts`:
       - Add Google avatar domain to `images.remotePatterns`:
         ```
         {
           protocol: "https",
           hostname: "lh3.googleusercontent.com",
         },
         ```
       - This allows `next/image` to load Google OAuth profile photos.
