# Phase 09.1: PWA Setup - Context

**Gathered:** 2026-03-02
**Status:** Ready for planning

<domain>
## Phase Boundary

Make Moodflix installable as a Progressive Web App — web app manifest, service worker, offline shell, caching, and install experience so users can add it to their home screen and get an app-like experience.

</domain>

<decisions>
## Implementation Decisions

### Install Experience
- New icon design — bold, cinematic, crimson & dark theme, optimized for home screens (192x192, 512x512, maskable variants)
- Branded loader splash screen — dark background + centered Moodflix logo + subtle animation
- Claude's discretion on install prompt UI (custom banner, in-app CTA, or browser default — pick what fits best)

### Offline Behavior
- Cached app shell + watchlist when offline — sidebar/nav loads, user can browse their cached library
- Subtle toast notification on connectivity change ("You're offline" / "Back online")
- Claude's discretion on whether offline mutations are read-only or queued
- Claude's discretion on whether AI mood section shows disabled state or hides entirely when offline

### Caching Strategy
- Pre-cache app shell (HTML, JS, CSS bundles) on service worker install — instant load on revisit
- Cache viewed TMDB images (posters, backdrops) — only what the user has actually seen, not pre-fetched
- Auto-cleanup with cache size limit to avoid filling device storage (Claude decides exact limit, ~50-100MB range)
- Claude's discretion on API response caching strategy per endpoint type (network-first vs stale-while-revalidate)

### Scope & Display Mode
- Standalone display mode — no browser chrome, app-like window
- Entire app in PWA scope — landing page, auth pages, and authenticated routes all within the PWA
- Any orientation supported — no portrait lock, responsive design handles rotation
- Claude's discretion on status bar / theme color (dark background vs crimson accent)

### Claude's Discretion
- Install prompt UI design and timing
- Offline mutation strategy (read-only vs queue-and-sync)
- AI mood section behavior when offline
- API response caching strategy per endpoint
- Status bar / theme color
- Cache size limit (within ~50-100MB range)
- Splash screen animation style

</decisions>

<specifics>
## Specific Ideas

- Icon should feel cinematic — crimson accent on dark background, bold enough to stand out at small sizes on any home screen wallpaper
- Splash screen should be a branded loader, not a mini landing page — fast and clean first impression
- Connectivity toast should be subtle (not modal or blocking) — brief appearance on state change

</specifics>
