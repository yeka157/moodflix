## Accomplishments

- Created `hooks/use-online-status.ts`: SSR-safe hook returning `isOnline` boolean from `navigator.onLine` + `online`/`offline` event listeners with cleanup
- Created `hooks/use-pwa-install.ts`: Captures `beforeinstallprompt` on Chromium browsers, exposes `canInstall` boolean and `install()` async function; returns `canInstall: false` on iOS/Firefox (progressive enhancement)
- Created `components/pwa/offline-toast.tsx`: Headless component that fires sonner `toast.success("Back online")` or `toast.warning("You're offline", { duration: Infinity, id: "offline-toast" })` on connectivity state change; `hasMounted` ref guard prevents firing on initial page load
- Created `components/pwa/install-prompt.tsx`: Fixed bottom banner (`bottom-16 md:bottom-0`) with `framer-motion` slide-up (`y: 100` → `y: 0`), `useReducedMotion()` guard, Install and dismiss (✕) buttons; hidden when `!canInstall || dismissed`
- Updated `app/(app)/layout.tsx`: Added `<OfflineToast />` and `<InstallPrompt />` after the main content area

## Files Created/Modified

- `hooks/use-online-status.ts` - SSR-safe navigator.onLine hook with window event listeners
- `hooks/use-pwa-install.ts` - BeforeInstallPromptEvent capture hook; Chrome/Android only via progressive enhancement
- `components/pwa/offline-toast.tsx` - Headless sonner toast component, hasMounted guard, offline persists (Infinity), online auto-dismisses

## Decisions Made

- Used `useRef(false)` hasMounted guard in OfflineToast to skip the effect on initial mount — prevents a "Back online" toast firing when the page loads while connected
- Install banner positioned `bottom-16 md:bottom-0` to sit above the mobile bottom tab bar (which is `pb-16` on mobile)
- `useReducedMotion()` from framer-motion sets `initial={{ y: 0 }}` when user prefers reduced motion — no slide-up animation
- Session-only dismiss via `useState` (no localStorage) — acceptable for MVP since the banner only appears after `beforeinstallprompt` fires anyway
- Used `<img>` (not Next.js `<Image>`) for the 24px icon in the install banner — fixed size with no dynamic sizing needed
