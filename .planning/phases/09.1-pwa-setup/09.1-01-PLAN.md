  <action>
Install Serwist packages:
```bash
npm install serwist
npm install -D @serwist/turbopack esbuild
```

Create `app/sw.ts` — the service worker source file. Use triple-slash directives for webworker types (do NOT add webworker to tsconfig.json lib). Configure:
- Precache entries from `self.__SW_MANIFEST` (injected by Serwist)
- `skipWaiting: true`, `clientsClaim: true`, `navigationPreload: true`
- Runtime caching rules (in order):
  1. TMDB images (`image.tmdb.org`): `CacheFirst`, cacheName `tmdb-images`, `maxEntries: 500`, `maxAgeSeconds: 30 * 24 * 60 * 60` (30 days), `CacheableResponsePlugin({ statuses: [0, 200] })`
  2. Own API routes (`/api/movies`, `/api/ai`): `NetworkFirst`, cacheName `api-routes`, `maxEntries: 50`, `maxAgeSeconds: 5 * 60` (5 min), `CacheableResponsePlugin({ statuses: [200] })`
  3. Spread `defaultCache` from `@serwist/turbopack/worker` for Next.js defaults
- Fallbacks: `{ entries: [{ url: "/~offline", matcher: ({ request }) => request.destination === "document" }] }`
- Call `serwist.addEventListeners()` at the end

Create `app/serwist/[path]/route.ts` — Turbopack route handler. Use `createSerwistRoute` from `@serwist/turbopack`:
- `revision`: Use `spawnSync("git", ["rev-parse", "HEAD"])` stdout, fallback to `crypto.randomUUID()`
- `additionalPrecacheEntries: [{ url: "/~offline", revision }]`
- `swSrc: "app/sw.ts"`
- `useNativeEsbuild: true`
- Export `dynamic`, `dynamicParams`, `revalidate`, `generateStaticParams`, `GET` from the created route

Reference the exact patterns from 09.1-RESEARCH.md Pattern 1 and Pattern 2.
