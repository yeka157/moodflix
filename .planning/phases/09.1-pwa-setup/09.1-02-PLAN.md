  <action>
**Create `hooks/use-online-status.ts`:**
- "use client" directive
- Returns `isOnline: boolean` from `navigator.onLine`
- SSR-safe: default to `true` when `typeof navigator === "undefined"`
- Registers `window.addEventListener("online" | "offline")` in useEffect, cleanup on unmount

**Create `hooks/use-pwa-install.ts`:**
- "use client" directive
- Define `BeforeInstallPromptEvent` interface extending `Event` with `prompt(): Promise<void>` and `userChoice: Promise<{ outcome: "accepted" | "dismissed" }>`
- useState for `deferredPrompt: BeforeInstallPromptEvent | null`
- useEffect: listen for `beforeinstallprompt`, `e.preventDefault()`, store event
- Expose `canInstall: boolean` (!!deferredPrompt) and `install: () => Promise<void>` (calls deferredPrompt.prompt(), clears state)
- This only fires on Chromium browsers — returns canInstall=false on iOS/Firefox (progressive enhancement)

**Create `components/pwa/offline-toast.tsx`:**
- "use client" directive
- Uses `useOnlineStatus()` hook
- Uses `useRef(false)` for `hasMounted` — skip toast on initial mount (prevent "Back online" toast on first load)
- useEffect watching `isOnline`:
  - If not mounted yet, set mounted and return
  - If online: `toast.success("Back online")` — auto-dismisses (default 4s)
  - If offline: `toast.warning("You're offline", { description: "Cached content is still available.", duration: Infinity, id: "offline-toast" })` — persists until back online
- Returns `null` (headless component, renders nothing)

Follow the exact patterns from 09.1-RESEARCH.md Pattern 4 (useOnlineStatus), Pattern 5 (usePwaInstall), and Code Example (OfflineToast).
