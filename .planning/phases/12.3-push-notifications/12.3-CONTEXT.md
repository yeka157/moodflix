# Phase 12.3: Push Notifications - Context

**Gathered:** 2026-03-07
**Status:** Ready for planning

<domain>
## Phase Boundary

Push notifications to alert users when upcoming movies from their watchlist are released. A bell icon on upcoming movie detail pages lets users opt in per title. A daily Vercel Cron job checks TMDB release dates against bell-subscribed items and sends Web Push notifications. Movies only — TMDB does not provide reliable upcoming dates for TV shows.

</domain>

<decisions>
## Implementation Decisions

### Push Technology
- Web Push API with VAPID keys — browser-native, no third-party dependency
- Integrates with existing Serwist service worker (`app/sw.ts`) for push event handling
- `web-push` npm package for server-side push delivery

### Bell Icon (Per-Title Opt-In)
- Bell icon appears on movie detail pages for **unreleased movies only**
- Tapping the bell subscribes/unsubscribes the user for release notifications on that specific title
- No global "notify all" toggle — per-title control only
- Bell icon only (no release date text alongside it)
- First bell tap auto-handles browser permission: if 'default' show browser prompt, if 'granted' subscribe silently, if 'denied' show toast explaining how to re-enable in browser settings

### Cron & Release Matching
- Vercel Cron job triggers an API route daily at 8 AM UTC
- Exact date match: `release_date == today (UTC)` — no window/fuzzy matching
- Protected with `CRON_SECRET` header — API route validates Authorization header to prevent unauthorized triggers
- Movies only — no TV show release notifications

### Notification Content
- Title + poster thumbnail from TMDB cache
- Notification title: "Now Available!" (or similar), body: movie title
- Tapping notification opens the movie detail page (`/movie/[id]`)
- Movies only scope (TMDB doesn't have reliable TV upcoming dates)

### Subscription Storage
- Two tables:
  1. `push_subscriptions` — Web Push subscription objects per user+device (endpoint, p256dh key, auth key)
  2. `notification_subscriptions` — per-user per-movie bell subscriptions (userId, tmdbId)
- Multi-device support: each device gets its own push subscription entry, cron sends to all active devices per user
- Device account switching: when a different user logs in on the same device, the `push_subscriptions` row's `userId` updates to the new user (keyed by endpoint)

### Claude's Discretion
- Notification grouping strategy (one per movie vs grouped) when multiple movies release same day
- Bell icon placement on detail page (action bar vs near release date)
- Stale subscription cleanup strategy (on failed delivery vs periodic)
- Service worker push event handler implementation details
- VAPID key generation and storage approach
- Bell icon visual design (filled vs outline toggle states)

</decisions>

## Existing Code Insights

### Reusable Assets
- `app/sw.ts`: Serwist service worker — extend with `push` and `notificationclick` event listeners
- `drizzle/schema.ts`: Database schema — add `push_subscriptions` and `notification_subscriptions` tables
- `lib/tmdb-cache.ts`: TMDB caching layer with `tmdb_media` table storing `release_date` — query for today's releases
- Movie detail page action bar: existing bookmark/watched/like icons — bell icon follows same pattern
- `lib/supabase/server.ts`: Server-side auth for identifying current user in API routes

### Established Patterns
- Fire-and-forget DB writes: `db.insert(...).catch(() => {})` for non-blocking operations
- Drizzle ORM for all DB operations with explicit userId WHERE clauses (RLS bypass)
- API route auth pattern: `createClient()` → `getUser()` → 401 if no user
- Action bar icons: Framer Motion hover/tap animations with spring physics (stiffness 200/25)
- `vercel.json` for deployment config — add cron schedule here

### Integration Points
- Movie detail page (`app/(app)/movie/[id]/page.tsx`): Add bell icon to action bar for unreleased movies
- Service worker (`app/sw.ts`): Add push event listener and notification click handler
- New API routes: `/api/cron/release-notifications` (cron target), `/api/push/subscribe` (subscription management)
- `vercel.json`: Add cron configuration for daily 8 AM UTC trigger
- `drizzle/schema.ts`: New tables for push and notification subscriptions

</code_context>
