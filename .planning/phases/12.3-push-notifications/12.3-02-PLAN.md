  <name>Task 1: Push subscription and notification toggle API routes</name>
  <files>app/api/push/subscribe/route.ts, app/api/notifications/subscribe/route.ts</files>
  <action>
1. Create `app/api/push/subscribe/route.ts`:

   **POST handler** - Save/update push subscription:
   - Authenticate user via `createClient()` + `getUser()`, return 401 if no user
   - Parse request body: `{ endpoint: string, keys: { p256dh: string, auth: string } }` (matches browser PushSubscription.toJSON() format)
   - Validate all fields present, return 400 if missing
   - Upsert into `pushSubscriptions`: ON CONFLICT on `endpoint`, update `userId` (handles device account switching per user decision)
   - Use `db.insert(...).onConflictDoUpdate({ target: pushSubscriptions.endpoint, set: { userId: user.id } })`
   - Return 200 with `{ success: true }`

   **DELETE handler** - Remove push subscription:
   - Authenticate user
   - Parse request body: `{ endpoint: string }`
   - Delete from `pushSubscriptions` WHERE endpoint matches AND userId matches
   - Return 200 with `{ success: true }`

2. Create `app/api/notifications/subscribe/route.ts`:

   **GET handler** - Check if user is subscribed to a specific movie:
   - Authenticate user
   - Read `tmdbId` from URL searchParams
   - Query `notificationSubscriptions` WHERE userId = user.id AND tmdbId = tmdbId
   - Return `{ subscribed: boolean }`

   **POST handler** - Subscribe to movie release notification:
   - Authenticate user
   - Parse body: `{ tmdbId: number, title: string, posterPath: string | null, releaseDate: string | null }`
   - Insert into `notificationSubscriptions` with ON CONFLICT DO NOTHING (idempotent)
   - Return 200 with `{ subscribed: true }`

   **DELETE handler** - Unsubscribe from movie release notification:
   - Authenticate user
   - Parse body: `{ tmdbId: number }`
   - Delete from `notificationSubscriptions` WHERE userId = user.id AND tmdbId = tmdbId
   - Return 200 with `{ subscribed: false }`

Both routes use explicit `eq(table.userId, user.id)` WHERE clauses (Drizzle bypasses RLS).

  <name>Task 2: Push subscription hook, bell icon component, and detail page integration</name>
  <files>hooks/use-push-subscription.ts, components/movies/bell-notify-button.tsx, components/movies/movie-detail-page.tsx, app/(app)/movie/[id]/page.tsx</files>
  <action>
1. Create `hooks/use-push-subscription.ts`:

   **Helper function** `urlBase64ToUint8Array(base64String: string): Uint8Array` (per Pitfall 1 in research):
   - Convert URL-safe Base64 VAPID key to Uint8Array for PushManager.subscribe()

   **`usePushSubscription()` hook:**
   - Returns `{ subscribe, unsubscribe, isSupported }`
   - `isSupported`: checks `"serviceWorker" in navigator && "PushManager" in window`
   - `subscribe()`: async function that:
     1. Checks `Notification.permission` -- if "denied", show toast "Notifications blocked. Enable in browser settings." and return null
     2. If "default", calls `Notification.requestPermission()` -- if not "granted", return null
     3. Gets `navigator.serviceWorker.ready`
     4. Calls `registration.pushManager.subscribe({ userVisibleOnly: true, applicationServerKey: urlBase64ToUint8Array(process.env.NEXT_PUBLIC_VAPID_PUBLIC_KEY!) })`
     5. Sends subscription to `/api/push/subscribe` via POST (with `subscription.toJSON()` body format: endpoint + keys.p256dh + keys.auth)
     6. Returns the PushSubscription object
   - `unsubscribe()`: async function that calls `subscription.unsubscribe()` then DELETE to `/api/push/subscribe`

   **`useNotificationSubscription(tmdbId: number)` hook** (uses TanStack Query):
   - `useQuery` to GET `/api/notifications/subscribe?tmdbId=${tmdbId}` -- returns `{ subscribed: boolean }`
   - `useMutation` for toggle: POST to subscribe, DELETE to unsubscribe
   - Optimistic updates on the query cache
   - Returns `{ isSubscribed, toggle, isLoading, isToggling }`

2. Create `components/movies/bell-notify-button.tsx`:

   ```
   "use client";
   ```

   **Props:** `{ tmdbId: number, title: string, posterPath: string | null, releaseDate: string | null }`

   - Uses `usePushSubscription()` for browser permission handling
   - Uses `useNotificationSubscription(tmdbId)` for per-movie state
   - On click:
     1. If not yet push-subscribed (first time), call `pushSubscription.subscribe()` to handle browser permission + save device subscription
     2. Then toggle notification subscription via `notificationSub.toggle()`
   - Icon: `Bell` (outline) when not subscribed, `BellRing` with `fill-current` + `text-primary` (crimson) when subscribed
   - Wrapped in `motion.div` with `whileTap={{ scale: 0.92 }}` (matches existing icon button pattern for small buttons)
   - `Button variant="ghost" size="icon"` with `aria-label="Notify me when released"` / `"Cancel release notification"`
   - Show `Loader2` spinner when isLoading or isToggling
   - If `!isSupported`, don't render (return null)
