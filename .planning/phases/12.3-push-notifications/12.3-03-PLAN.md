  <action>
Create `app/api/cron/release-notifications/route.ts` with a single GET handler:

1. **Auth guard** (per Vercel Cron docs):
   ```typescript
   const authHeader = request.headers.get("authorization");
   if (authHeader !== `Bearer ${process.env.CRON_SECRET}`) {
     return new Response("Unauthorized", { status: 401 });
   }
   ```

2. **Get today's date** in UTC as YYYY-MM-DD string:
   ```typescript
   const today = new Date().toISOString().slice(0, 10);
   ```

3. **Find movies releasing today** -- join `notificationSubscriptions` with `tmdbMedia`:
   - Query: SELECT notificationSubscriptions.userId, notificationSubscriptions.tmdbId, notificationSubscriptions.title (fallback), tmdbMedia.title, tmdbMedia.posterPath
   - INNER JOIN tmdbMedia ON (tmdbMedia.tmdbId = notificationSubscriptions.tmdbId AND tmdbMedia.mediaType = 'movie')
   - WHERE tmdbMedia.releaseDate = today
   - AND (notificationSubscriptions.lastNotifiedAt IS NULL OR notificationSubscriptions.lastNotifiedAt < start of today) -- idempotency guard (Pitfall 6)
   - Use Drizzle `isNull` or `lt` for the lastNotifiedAt check

   If no results, return early: `{ success: true, notified: 0, movies: 0 }`

4. **Also handle the case where tmdb_media row doesn't exist** (Pitfall 7):
   - For notification subscriptions where the tmdbId has no tmdb_media row, fall back to the `releaseDate` cached in the notification_subscriptions row itself
   - Query: SELECT from notificationSubscriptions WHERE releaseDate = today AND tmdbId NOT IN (tmdbIds already found above) AND (lastNotifiedAt IS NULL OR lastNotifiedAt < today)
   - This catches movies the user subscribed to but never revisited (so no tmdb_media cache entry)

5. **Group by userId** -- collect all movies per user

6. **For each user, get all their push subscriptions:**
   - Query pushSubscriptions WHERE userId = user's id
   - If no push subscriptions, skip (user removed browser data)

7. **Send notifications** -- one notification per movie (not grouped, per discretionary recommendation):
   - For each movie per user, for each device:
     ```typescript
     const payload: PushPayload = {
       title: "Now Available!",
       body: movieTitle,
       url: `/movie/${tmdbId}`,
       icon: "/icons/icon-192x192.png",
       badge: "/icons/icon-72x72.png",
     };
     ```
   - Use `Promise.allSettled()` for batch delivery (per anti-pattern guidance -- never loop synchronously)
   - For each settled result, if rejected with statusCode 410 or 404, delete that push subscription row (stale cleanup per Pitfall 2):
     ```typescript
     // Import WebPushError from web-push for type checking
     if (error.statusCode === 410 || error.statusCode === 404) {
       await db.delete(pushSubscriptions).where(eq(pushSubscriptions.endpoint, endpoint));
     }
     ```

8. **Update lastNotifiedAt** for all successfully notified notification_subscriptions rows:
   - `db.update(notificationSubscriptions).set({ lastNotifiedAt: new Date() }).where(...)` for each notified (userId, tmdbId) pair

9. **Return summary:**
   ```typescript
   return Response.json({
     success: true,
     movies: uniqueMovieCount,
     users: uniqueUserCount,
     notified: totalNotificationsSent,
     staleRemoved: staleSubscriptionsRemoved,
   });
   ```
