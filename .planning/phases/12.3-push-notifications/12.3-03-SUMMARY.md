# Phase 12.3 Plan 03: Cron Release Notifications Summary

**Daily cron route with TMDB cache join, batch push delivery via Promise.allSettled, stale cleanup, and idempotency guard**

## Accomplishments
- Cron route authenticates via CRON_SECRET Bearer header (Vercel Cron pattern)
- Joins notification_subscriptions with tmdb_media for today's releases, with fallback for missing cache entries
- Batch delivery via Promise.allSettled with stale subscription cleanup on 410/404
- Idempotency via lastNotifiedAt prevents duplicate notifications on re-trigger
- Lazy VAPID initialization fixes build-time crash when env vars unavailable

## Decisions Made
- Lazy VAPID initialization: `setVapidDetails` moved from module-level to `ensureVapid()` function called before each send -- prevents build-time crash when VAPID env vars are undefined
- Individual notification per movie (not grouped) for clearer mobile notification UX
- Fallback query for notification_subscriptions.releaseDate when no tmdb_media row exists (covers unvisited movies)
