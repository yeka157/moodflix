# External Integrations

**Analysis Date:** 2026-02-17

## APIs & External Services

**Movie Data:**
- TMDB (The Movie Database) - Search, trending, popular, top-rated, genre discovery, watch providers
  - SDK/Client: `lib/tmdb.ts` (native fetch + Bearer token)
  - Auth: `TMDB_API_READ_KEY` (v4 API Bearer token)
  - Endpoints:
    - `/trending/movie/week` - Trending movies this week
    - `/movie/popular` - Popular movies by region
    - `/movie/top_rated` - Top-rated movies
    - `/search/movie` - Full-text search
    - `/discover/movie` - Filter by genre, sort by popularity
    - `/movie/{id}` - Details with `append_to_response=credits,watch/providers` (cast, crew, regional watch providers)
    - `/movie/{id}/recommendations` - Recommended movies by TMDB algorithm
  - Rate Limiting: Standard TMDB rate limits apply
  - Caching: ISR 5 minutes on all requests (Next.js `next: { revalidate: 300 }`)

**AI Recommendations:**
- Google Generative AI (Gemini) - Mood-based movie recommendations via streaming chat
  - SDK/Client: `@ai-sdk/google` via `lib/ai.ts` (Vercel AI SDK wrapper)
  - Model: `gemini-2.0-flash` (streaming capable, structured output via tool calling)
  - Auth: `GOOGLE_GENERATIVE_AI_API_KEY` (API key)
  - Endpoint: `app/api/ai/recommend/route.ts` (POST /api/ai/recommend)
  - Features:
    - Streaming text responses (real-time chat)
    - Tool calling: `suggest_genres` tool with structured output
    - System prompt includes user's watchlist context for personalization
    - Max input: 500 characters (validated)
    - Max output: 1000 tokens per response
    - Multi-turn conversation via UIMessages format
  - Rate Limiting: 10 requests/day per user (LRU in-memory cache in `lib/rate-limit.ts`)
  - Response Type: `application/stream+json` (Server-Sent Events via `toUIMessageStreamResponse()`)

## Data Storage

**Databases:**
- PostgreSQL (Supabase) - All persistent data (users, watchlist, recommendations)
  - Connection URLs:
    - `DATABASE_URL` - Transaction pooler (port 6543, pgBouncer) for runtime queries
    - `DATABASE_URL_DIRECT` - Session pooler (port 5432, no pgBouncer) for Drizzle Kit migrations
  - ORM: Drizzle ORM with `postgres` driver (native Node.js)
  - Tables:
    - `profiles` - User profiles extending auth.users (id, username, avatarUrl, createdAt)
    - `watchlist` - User movie watchlist (id, userId, tmdbId, title, posterPath, status, rating, addedAt, watchedAt)
      - Status enum: `want_to_watch`, `watching`, `watched`
      - Unique constraint: (userId, tmdbId) — one entry per user per movie
    - `ai_recommendations` - Stored AI recommendations (id, userId, prompt, recommendations JSON, createdAt)
  - RLS (Row Level Security): Enabled on all tables, policies managed via Supabase Dashboard
  - Migrations: Generated by Drizzle Kit → `drizzle/migrations/` (applied via `npm run db:migrate`)

**File Storage:**
- Not implemented — placeholder images used locally
  - Fallbacks: `public/placeholder-poster.svg`, `public/placeholder-backdrop.svg`
  - Future: Could integrate Supabase Storage or AWS S3 for user avatars

**Caching:**
- In-Memory (LRU Cache) - Rate limiting per user in `lib/rate-limit.ts`
  - Library: `lru-cache` (TTL: 24 hours)
  - Stores request count per user ID for AI recommendations
- Next.js Data Cache - ISR for TMDB API responses
  - Revalidate: 300 seconds (5 minutes)
- TanStack Query Client-Side Cache - Movie search, trending, popular results
  - Managed in `lib/query-client.ts`
  - Stale time, cache time configurable per query hook

## Authentication & Identity

**Auth Provider:**
- Supabase Auth (custom OAuth + email/password + passkey support)
  - Implementation: `@supabase/ssr` (browser & server clients)
  - Email/Password Auth:
    - Sign up: `actions/auth.ts` → `signup()` server action
    - Sign in: `actions/auth.ts` → `login()` server action
    - No email confirmation required (immediate redirect to `/home` after signup)
  - OAuth Providers:
    - Google OAuth via `loginWithGoogle()` server action
    - Callback route: `app/(auth)/callback/route.ts` (handles OAuth redirect)
  - Passkey Support: Configured in Supabase, user-initiated enrollment (not forced)
  - Session Management:
    - Browser client: `lib/supabase/client.ts` (createBrowserClient)
    - Server client: `lib/supabase/server.ts` (createServerClient with cookie handler)
    - Middleware refresh: `lib/supabase/middleware.ts` (updateSession on every request)
  - Logout: `actions/auth.ts` → `logout()` server action
  - Route Protection:
    - Protected routes: `/home`, `/discover`, `/watchlist` (redirect to `/login` if not authenticated)
    - Auth pages: `/login`, `/signup`, `/callback` (redirect to `/home` if authenticated)

## API Route Implementation

**Movie Proxy Routes:**
- `app/api/movies/route.ts` (GET)
  - Query params: `query` (search), `category` (trending|popular|top_rated), `genre` (genre IDs), `page`
  - Proxies to TMDB, handles caching, returns JSON
- `app/api/movies/[id]/route.ts` (GET)
  - Path param: `id` (TMDB movie ID)
  - Returns movie details + regional watch providers
  - Country detection via `x-vercel-ip-country` header (defaults to US)
  - Watch providers for detected country injected as `watchProviders` in response

**AI Recommendation Route:**
- `app/api/ai/recommend/route.ts` (POST)
  - Body: `{ messages: UIMessage[] }` (AI SDK message format)
  - Auth: Requires Supabase session (401 if missing)
  - Rate limit: 10 requests/day per user (429 with `Retry-After` header)
  - Validation: Last message max 500 characters
  - Response: Streaming JSON with text chunks and tool results
  - Tool: `suggest_genres` generates TMDB genre recommendations and stores in `ai_recommendations` table
